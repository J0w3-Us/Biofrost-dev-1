# Apple Human Interface Guidelines - Biofrost Adaptation

Este documento sirve como la base para todas las decisiones de diseño y UX/UI en el proyecto Biofrost, asegurando consistencia y la "calidad Apple".

## 1. Principios de Diseño
- **Estética de la Integridad**: El diseño debe verse y sentirse como una extensión natural del sistema operativo.
- **Consistencia**: Usar elementos comunes y comportamientos predecibles.
- **Control Directo**: Los usuarios deben sentir que tienen el control de la aplicación, no al revés.
- **Retroalimentación**: Confirmar acciones con respuestas visuales o hápticas sutiles.
- **Metáforas**: Usar conceptos familiares del mundo real para facilitar el aprendizaje.

## 2. Experiencia de Usuario (UX)
- **Minimalismo con Propósito**: Eliminar el desorden visual; cada elemento debe tener una función.
- **Navegación Intuitiva**: Estructuras claras de navegación (Tab Bars, Navigation Bars) que no confundan al usuario.
- **Eficiencia**: Reducir el número de pasos para completar tareas clave.

## 3. Interfaz de Usuario (UI)
- **Tipografía**: Priorizar la legibilidad y el uso de jerarquías claras (ej. San Francisco o equivalentes).
- **Espaciado y Aire**: Uso generoso de espacios en blanco para evitar la saturación.
- **Color**: Paletas armoniosas, soporte nativo para Dark Mode y contrastes de accesibilidad.
- **Micro-animaciones**: Transiciones fluidas y orgánicas que den vida a la interfaz.

## 4. Reglas Críticas
- **No Placeholders**: Todas las pantallas deben representar datos reales o estados de carga elegantes.
- **Consistencia de Bordes**: Usar radios de esquina consistentes (estilo Apple "squircle").
- **Interactividad**: Todo elemento interactivo debe tener un estado visual claro (hover, tap, active).
